Pourquoi
L'alphabet visuel de l'interface
Boutons, champs, badges, avatars — ce sont les atomes que chaque page de CMS SARIS Congo assemble. Leur cohérence rigoureuse garantit que les soignants reconnaissent instantanément l'action à effectuer, l'état du système, et l'identité de chaque personne dans un dossier patient.
Quoi
6 familles de composants atomiques
Boutons (5 variants × 3 tailles × 5 états) · Champs de formulaire (input, textarea, select, checkbox, radio, toggle) · Badges & tags sémantiques · Avatars avec indicateurs de présence · Séparateurs horizontaux/verticaux/étiquetés · Tooltips (4 positions).
Comment
Tokens + états CSS + grain sur solids
Toutes les couleurs via tokens. Grain SVG (::after opacity 0.38) sur boutons solides uniquement. Focus visible via outline offset. Chaque état (hover, focus, active, disabled, loading) est explicitement défini et rendu ici.
Section 01
Boutons

Section 03
Badges & Tags
Les badges communiquent un statut ou une catégorie. Les tags représentent des métadonnées attachées à un objet (patient, dossier) et peuvent être retirés. Grain discret (opacity 0.18) sur les badges colorés. Border-radius xs=4px obligatoire — aucun badge pill.

Section 06
Tooltips
Fond sombre en mode clair (--g-900), fond moyen avec bordure en mode sombre (--g-800). 4 positions (top, bottom, left, right). Caret triangulaire de 5px. Apparition à 150ms. Texte court, jamais plus de 2 lignes.

Règles absolues
Obligations & Interdictions — Composants atomiques
Ces règles s'imposent à tous les développeurs et agents IA sur CMS SARIS Congo, sans exception.
Obligations
✓Grain obligatoire sur tous les boutons solid via ::after opacity 0.38 mix-blend-mode overlay.
✓Tous les champs de formulaire utilisent --fond-input comme fond et --radius-xs comme rayon.
✓Le focus-visible est toujours stylisé — outline 2px couleur d'accent + offset 3px — pour l'accessibilité clavier.
✓Les badges colorés portent le grain à opacity 0.18 maximum. Les tags sont toujours en fond neutre, jamais colorés.
✓Les avatars sont toujours circulaires (border-radius 50%). Les initiales sont en majuscules, police Inter 600.
✓Les tooltips apparaissent après 400ms de hover. Jamais d'information critique uniquement en tooltip — elle doit aussi exister dans le contenu.
Interdictions absolues
✗Aucun border-radius supérieur à 10px (--radius-lg) sur les boutons. Zéro bouton pill (border-radius 999px) dans toute l'interface.
✗Aucune couleur hardcodée — ni dans les boutons, ni dans les badges, ni dans les avatars. Tout via tokens CSS.
✗Pas de cursor: not-allowed sans désactivation visuelle (opacity 0.40). Les deux vont ensemble.
✗Pas d'emoji ou d'icône ASCII dans les boutons, badges ou labels. Uniquement des SVG Lucide.
✗Les tooltips ne contiennent jamais d'éléments interactifs (liens, boutons). Ils sont en lecture seule.
✗Aucun badge avec border-radius 999px. Les badges médico-sociaux restent discrets et précis — 4px uniquement.
