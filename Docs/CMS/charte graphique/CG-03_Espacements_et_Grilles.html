Pourquoi

La respiration crée la clarté

Dans un CMS médical dense en données, l'espacement n'est pas du luxe — c'est un outil de sécurité. Un bon espacement réduit les erreurs de lecture entre deux lignes de tableau, distingue clairement les blocs de données, et guide l'œil sans effort.

Quoi

Unité de base 4px · échelle ×2 jusqu'à 64px · style aéré Linear

Tout espacement est un multiple de 4px. L'échelle couvre 12 niveaux de 2px à 64px. Le style général est aéré (style Linear/Vercel) : on préfère trop de respiration à pas assez. La grille de contenu est à 12 colonnes avec gouttières de 24px.

Comment

Tokens --espace-N · zones de layout nommées

Chaque valeur d'espacement est un token CSS (--espace-1 à --espace-16). Les zones du layout (sidebar, topbar, contenu, padding de page) ont toutes une valeur fixe nommée. Un développeur n'écrit jamais une valeur brute en px — il utilise le token.

Section 05

Padding & Densité des composants — Valeurs de référence

Chaque type de composant a un padding standardisé. Ces valeurs garantissent la cohérence visuelle entre tous les écrans. Le style est aéré — en cas de doute, on choisit toujours la valeur plus généreuse.

ComposantPadding verticalPadding horizontalToken(s)Justification
Page de contenu 48px 40px --espace-12 / --espace-10 Respiration généreuse — le contenu ne colle jamais aux bords
Carte standard 20px 24px --espace-5 / --espace-6 Équilibre entre densité et lisibilité pour les cartes de données
Carte compacte (KPI) 14px 16px --espace-3.5 / --espace-4 Cartes de statistiques sur tableau de bord — densité acceptable
Cellule de tableau 10px 14px --espace-2.5 / --espace-3.5 Densité optimisée pour les tableaux de données — lisible sans être trop compact
Bouton standard 7px 16px --espace-1.75 / --espace-4 Hauteur perçue ~36px — confortable sans être massif
Bouton compact 4px 10px --espace-1 / --espace-2.5 Boutons dans les tableaux ou en contexte dense — hauteur ~28px
Champ de formulaire 7px 12px --espace-1.75 / --espace-3 Cohérent avec le bouton standard — ligne de base alignée
Item de navigation sidebar 6px 10px --espace-1.5 / --espace-2.5 Dense mais lisible — permet d'afficher 8 à 10 items sans scroll
Header de section 0 bas · 16px bas — --espace-4 Séparation claire entre le titre de section et le contenu

Sidebar (padding interne) 10px 10px --espace-2.5 Dense — la sidebar doit contenir beaucoup d'items sans scroller

Section 06

Border-radius — Arrondis discrets

Les arrondis sont discrets et cohérents. Le principe est simple : plus un composant est petit, plus son arrondi est petit. Aucun style "pill" généralisé. Pas d'arrondi supérieur à 12px dans l'interface fonctionnelle.

50%

Avatars uniquement · indicateurs ronds · jamais pour les boutons

Règle absolue : le border-radius minimum sur tout bouton est 4px (--radius-xs). Le style "pill" (border-radius: 9999px) est interdit sur les boutons — il n'est autorisé que sur les avatars et indicateurs circulaires.

Section 07

Règles absolues — Obligations & Interdictions

Ces règles s'appliquent à tous les composants et layouts du CMS SARIS Congo.

Obligations

✓Tout espacement est un multiple de 4px — utiliser les tokens --espace-N. Jamais de valeur brute en px dans le code.
✓Le style d'espacement est aéré (style Linear) — en cas de doute entre deux valeurs, choisir la plus généreuse.
✓La sidebar est fixée à 240px. La topbar à 54px. Le contenu prend le reste. Ces dimensions ne changent pas selon le module.
✓La gouttière de grille est toujours 24px (--espace-6) entre les colonnes de contenu.
✓Le max-width du contenu est 1100px — le contenu est centré avec des marges auto au-delà.
✓Le border-radius minimum sur tout bouton est 4px. Les avatars sont les seuls éléments en border-radius: 50%.

Interdictions

✗Aucune valeur d'espacement hors de l'échelle (ex : 5px, 7px, 11px, 15px…) — uniquement les multiples de 4.
✗Aucun style "pill" (border-radius: 9999px) sur les boutons — uniquement les avatars.
✗Aucun border-radius supérieur à 12px dans l'interface fonctionnelle.
✗Aucune largeur fixe pour la zone de contenu — elle est toujours calculée en soustrayant la sidebar.
✗Ne jamais coller le contenu aux bords de la zone sans padding — le minimum est 24px de chaque côté.
✗Aucune grille de plus de 4 colonnes pour les cartes KPI — au-delà, passer à 2 rangées de 4 ou réduire les colonnes.
