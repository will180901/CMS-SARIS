Pourquoi
Mouvement qui rassure
Dans un CMS médical, chaque geste de l'utilisateur doit recevoir une confirmation immédiate. Une animation bien calibrée réduit l'anxiété de l'utilisateur face à une action critique — suppression d'un dossier, envoi d'une ordonnance, synchronisation entre Moutela et Nkayi.
Quoi
5 courbes · 6 durées · Micro-interactions
Cinq fonctions de timing couvrent tous les cas (standard, decelerate, accelerate, spring, spring-soft). Six paliers de durée de 80 ms à 400 ms. Un catalogue complet des micro-interactions et des transitions par composant.
Comment
Tokens uniquement, jamais de valeurs brutes
Toutes les animations utilisent les tokens --ease-* et --dur-*. Jamais de 0.3s ease hardcodé. La règle absolue : aucune animation ne dure plus de 400 ms et toutes respectent prefers-reduced-motion.

Section 05

Règles absolues d'animation

Six règles non négociables qui garantissent une interface qui répond sans jamais distraire l'utilisateur soignant.

Interdictions absolues
❌ Interdit
Jamais de valeur de durée brute. Ne pas écrire 0.3s ease ou 300ms linear. Toujours utiliser les tokens --dur-* et --ease-*.
✓ Correct
transition: all var(--dur-rapide) var(--ease-standard); — maintenable, modifiable globalement depuis les tokens.
❌ Interdit
Jamais d'animation > 400 ms. Au-delà, l'interface paraît lente. Même les transitions de page restent en deçà de 400 ms.
✓ Correct
La durée maximale est --dur-tres-lent: 400ms — réservée aux accordéons et transitions de page uniquement.
❌ Interdit
Jamais d'animation sur tous les éléments à la fois. * { transition: all 0.3s } génère des transitions indésirables (scroll, resize, focus).
✓ Correct
Cibler uniquement les propriétés concernées : transition: background var(--dur-instant), border-color var(--dur-instant);
Règles de performance & accessibilité
Respecter prefers-reduced-motion
Toujours inclure une media query qui désactive les animations pour les utilisateurs qui la demandent. Les effets de fondu (opacity) peuvent rester, pas les translations ni les transformations complexes. @media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; } }
Animer uniquement transform et opacity
Ces deux propriétés sont composited par le GPU — aucun reflow, aucun repaint. Éviter d'animer width, height, top, margin sauf si absolument nécessaire (accordéons).
Entrée ≠ Sortie
Les modales et drawers entrent en --ease-decelerate (arrivée rapide, arrêt doux) et sortent en --ease-accelerate (sortie franche). La même courbe dans les deux sens brise le sentiment de profondeur.
