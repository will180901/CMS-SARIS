Pourquoi

Une typographie au service de la lisibilité médicale

Dans un CMS médico-social, la typographie n'est pas décorative — elle porte des données critiques (noms de patients, dosages, dates de permutation). Elle doit être parfaitement lisible à toutes les tailles, sur tous les écrans, sans jamais fatiguer l'œil.

Quoi

Inter · Humaniste · Deux graisses · Onze niveaux

La police Inter (Rasmus Andersson) est la référence de Linear, GitHub et Vercel. Humaniste et optimisée pour les écrans, elle offre une lisibilité exceptionnelle à 11px comme à 28px. Le système utilise uniquement deux graisses : 400 (régulier) et 500 (medium). Onze niveaux de taille couvrent tous les contextes.

Comment

Tokens nommés · échelle proportionnelle · interlignage contextuel

Chaque niveau est un token CSS (--typo-display, --typo-h1…). La taille de base est 14px. L'interlignage varie selon le niveau : plus le texte est grand, plus l'interlignage est serré. Aucun texte ne descend en dessous de 11px. Aucun titre n'est surdimensionné.

Section 01

Famille de police — Inter

Inter est chargée depuis Google Fonts avec les graisses 400 et 500 uniquement. La graisse 600 est autorisée uniquement pour les numéros de badge et labels de navigation — elle doit rester exceptionnelle. Aucune autre famille de police n'est utilisée dans l'interface (à l'exception du monospace pour les tokens et codes).

Section 02

Échelle typographique — 11 niveaux

Chaque niveau a une taille, une graisse, un interlignage et un espacement de lettres précis. Les niveaux sont conçus pour créer une hiérarchie claire sans jamais utiliser de grands titres imposants. La taille maximale (display) est 28px — réservée aux métriques de tableau de bord.

Section 04

Couleurs typographiques — Hiérarchie par couleur

La couleur renforce la hiérarchie. Trois niveaux de gris + une couleur d'accent couvrent tous les contextes. La couleur sémantique s'ajoute pour les états d'erreur, succès et avertissement dans les labels de formulaire.

Texte primaire — --texte-primaire

Valeurs, titres, noms de patients, données critiques, contenu principal de tableau

Clair : #0F172A · Sombre : #E6EDF3

Texte secondaire — --texte-secondaire

Descriptions, sous-titres, métadonnées, textes d'accompagnement, valeurs moins critiques

Clair : #475569 · Sombre : #8B949E

Texte tertiaire — --texte-tertiaire

Captions, placeholders, labels discrets, horodatages, overlines de navigation

Clair : #94A3B8 · Sombre : #484F58

Texte accent — --couleur-accent

Liens actifs, actions cliquables, valeurs d'accent dans les tableaux, texte de badge primaire

Identique : #4E8BA4 dans les deux modes

Texte sémantique — ex. --erreur-texte

Labels de champs en erreur · messages de validation · titres d'alertes inline. Utiliser la couleur sémantique du contexte (succes / erreur / avert / info).

Section 07

Règles absolues — Obligations & Interdictions typographiques

Ces règles s'appliquent à tous les textes de l'interface CMS SARIS Congo, sans exception.

Obligations

✓Toujours utiliser Inter — jamais d'autre police dans l'interface fonctionnelle.
✓Uniquement les graisses 400 et 500 dans le corps de l'interface. 600 uniquement pour les overlines et badges de navigation.
✓Taille minimale : 11px pour tout texte fonctionnel visible et interactif.
✓Taille maximale : 28px (display) — uniquement pour les valeurs de KPI.
✓Le monospace (Courier New) est réservé aux tokens, codes de dossier, numéros de bon de pharmacie, et identifiants techniques.
✓L'overline (10px, uppercase, letter-spacing) est toujours utilisée pour les titres de groupe dans la sidebar — jamais pour du contenu.
✓Appliquer le rendu antialiased : -webkit-font-smoothing: antialiased sur le body.

Interdictions

✗Aucun texte en dessous de 11px — même pour les footnotes ou annotations techniques.
✗Aucune graisse 700 (bold) dans l'interface — elle casse la cohérence visuelle d'Inter.
✗Aucun texte en MAJUSCULES dans le corps (corps de tableau, descriptions, noms) — uniquement autorisé pour les overlines de navigation.
✗Aucun titre de page supérieur à 22px — pas de grands titres imposants style "hero".
✗Aucun espacement de lettres (letter-spacing) sur le corps de texte et les titres H1–H4 — uniquement sur overlines et labels.
✗Aucune italique dans les tableaux de données ou formulaires — l'italique est réservé aux citations et notes cliniques longues.
✗Ne jamais justifier le texte (text-align: justify) — toujours left-aligned ou start.
