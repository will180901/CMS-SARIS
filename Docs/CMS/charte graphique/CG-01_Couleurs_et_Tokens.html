Pourquoi

Un système ancré dans l'identité de marque

Toutes les couleurs de l'interface sont directement extraites du logo SARIS Congo. Ce choix garantit une cohérence totale entre la marque institutionnelle et le logiciel, renforçant la légitimité du CMS auprès des équipes médicales de Moutela et Nkayi.

Quoi

Bleu ardoise primaire · Or Congo secondaire · Gris froids

Le bleu ardoise (#4E8BA4) incarne confiance, rigueur et calme médical. L'or Congo (#C99A2E) apporte singularité, réservé aux éléments de mise en valeur. Les fonds sont des gris froids bleutés style GitHub — sérieux et neutres.

Comment

Tokens CSS nommés · nuances 50 → 900 · double mode

Chaque couleur est un token CSS avec 10 nuances. Les tokens contextuels (--fond-page, --couleur-accent…) s'adaptent automatiquement au mode clair et sombre. Aucune valeur hex ne doit être hardcodée dans les composants — on utilise toujours les tokens.

Section 05

Couleurs sémantiques — États & Signaux

Volontairement désaturées pour s'intégrer au monochrome sans le dominer. Chaque couleur sémantique existe en quatre rôles indissociables : fond, texte/icône, accent (bordure gauche ou icône forte), bordure. Ces quatre tokens sont toujours utilisés ensemble — jamais séparément.

Succès

Action validée · donnée confirmée · stock suffisant · permutation confirmée

--succes-fond#EAF3EC
--succes-texte#2D6A3F
--succes-accent#3D7A5E
--succes-bordurergba(45,106,63,.2)

Erreur

Champ invalide · action échouée · rupture critique · accès refusé

--erreur-fond#F5EAEA
--erreur-texte#7A2E2E
--erreur-accent#9B4444
--erreur-bordurergba(122,46,46,.2)

Stock faible · délai dépassé · bon non signé · donnée à vérifier

--avert-fond#F5EEDF
--avert-texte#6B4D1A
--avert-accent#8A6A2A
--avert-bordurergba(107,77,26,.2)

--info-fond#E6EFF5
--info-texte#2A4F6B

Mode sombre : les couleurs sémantiques sont recalibrées — fonds très sombres, textes désaturés plus clairs. Les quatre tokens s'adaptent automatiquement via les variables CSS du thème.

Section 09

Règles absolues — Obligations & Interdictions

Ces règles s'appliquent sans exception à tous les composants du CMS SARIS Congo. Toute déviation constitue une violation de la charte graphique.

Obligations — à respecter toujours

✓Toujours utiliser les tokens CSS (--fond-page, --couleur-accent…) — jamais de valeur hex hardcodée dans un composant.
✓Texte sur fond accent (#4E8BA4) : toujours blanc #FFFFFF — ratio WCAG 4.5:1 minimum.
✓Texte sur fond Or (#C99A2E) : toujours la nuance 900 (#4D3A08) — jamais blanc, jamais noir pur.
✓Les fonds de page et de surface sont toujours des gris froids bleutés — jamais de blanc #FFFFFF comme fond de page en mode clair.
✓Les quatre tokens sémantiques (fond / texte / accent / bordure) sont toujours utilisés ensemble pour un même état.
✓En mode sombre, les bordures sont toujours en rgba blanc avec opacité — jamais en valeur hex grise fixe.
✓Les couleurs sémantiques (succès, erreur, avertissement, info) restent toujours désaturées — jamais de rouge #FF0000 ou vert #00FF00.

Interdictions — à ne jamais faire

✗Aucun dégradé sur boutons, cartes, fonds de page ou tout composant d'interface courant. Dégradé autorisé uniquement sous les courbes de graphiques statistiques (transparent, discret).
✗Les couleurs d'accent (bleu ou or) ne servent jamais de fond de page ou de surface — uniquement en accent.
✗Ne jamais mélanger bleu ardoise et or Congo sur un même composant (ex : bouton bicolore) — ils s'utilisent en alternance.
✗Les couleurs de marque brutes (Brun SARIS #6B3F26, Sable chaud #8A7460) n'apparaissent jamais dans l'UI fonctionnelle.
✗Aucun fond blanc pur #FFFFFF comme fond de page en mode clair — utiliser --fond-page (#F4F6F8).
✗Aucun gris chaud (beige, crème, ivoire) — tous les gris tirent vers le bleu froid.
✗Aucune couleur sémantique vive non désaturée — toujours utiliser les versions définies dans ce module.
